# afuOS - 本地优先的 macOS AI 桌面助手

[English README](README.md)

afuOS 是一款本地优先的 macOS AI 桌面助手，基于 Tauri、Rust、React 和 TypeScript 构建。

它提供一个轻量的悬浮助手窗口，支持文本、语音和图片交互、OpenAI 兼容聊天、本地桌面操作、语音播放、会话历史、执行日志和用户管理的长期记忆。

你可以把 afuOS 当作私有的 Mac AI 助手，用于日常桌面自动化、多模态聊天、语音输入、语音输出、技能加载、MCP 工具调用，以及带权限控制的本地操作。

## 设计理念

afuOS 的灵感来自《蝙蝠侠》里随叫随到的阿福管家。它希望把这种“需要时立刻出现”的助手体验带给普通 Mac 用户。

它像手机助手一样容易唤醒，但在桌面场景中更智能、更强大。afuOS 可以记录你的偏好，理解你的工作上下文，并帮助你处理实际任务，而不只是回答一次性问题。

## 功能特性

- 悬浮助手窗口，支持全局快捷键唤醒和隐藏。
- OpenAI 兼容的聊天流式输出。
- 支持为文本、视觉、语音合成和语音识别配置不同模型档案。
- 聊天图片附件。用户图片会作为 OpenAI 兼容的 `image_url` 内容发送到选中的视觉模型档案，助手回复可以渲染 Markdown、远程、本地和 data URL 图片。
- 支持 macOS 原生语音识别或基于模型的语音识别。
- 可选的文本转语音回复。
- 本地操作支持打开应用、URL、文件、文件夹、备忘录、提醒事项、剪贴板文本和经确认的 shell 命令。
- 本地操作权限控制。低风险 shell 和浏览器操作可以在设置中自动允许，也可以按目标记住授权；高风险操作始终需要明确确认。
- 禁区路径会覆盖已保存的权限规则，适用于文件、shell、`file://`、递归搜索和 MCP 访问。
- Skill 和 MCP 注册表支持启用和禁用。受信任的 skill 可以注入本地 `SKILL.md` 内容；未受信任的 skill 只暴露元数据；MCP 服务器可以在权限检查下检查和调用。
- 本地 SQLite 存储会话历史、图片附件、执行日志、权限规则和记忆记录。
- 可编辑记忆文件和助手人格档案文件。
- API key 通过 macOS Keychain 存储，不写入明文配置文件。

## 使用场景

- 运行一个本地优先的 macOS AI 助手，桌面数据只会发送到你配置的模型端点。
- 在紧凑的悬浮窗口中使用 OpenAI 兼容文本模型和视觉模型聊天。
- 使用语音输入、macOS 原生语音识别、模型转写和文本转语音回复。
- 执行带权限控制的 Mac 自动化，包括应用、URL、文件、文件夹、备忘录、提醒事项、剪贴板文本、shell 命令和 MCP 工具。
- 将私有会话历史、记忆、图片元数据、权限规则和执行日志保存在本地 SQLite 中。
- 加载受信任的本地 `SKILL.md` 文件，并连接 MCP 服务器来支持工具型 AI 工作流。

## 环境要求

- macOS
- Node.js 20+

- Xcode Command Line Tools

## 开发

安装依赖：

```bash
npm install
```

以开发模式运行桌面应用：

```bash
npm run tauri -- dev
```

构建前端：

```bash
npm run build
```

运行 Rust 测试：

```bash
cd src-tauri
cargo test
```

格式化 Rust 源码：

```bash
cd src-tauri
cargo fmt
```

构建 macOS app bundle：

```bash
npm run tauri -- build --bundles app
```

生成的 app bundle 位于：

```text
src-tauri/target/release/bundle/macos/afuos.app
```

## 配置

模型设置在应用内配置。API key 保存在 macOS Keychain 中。本地配置文件只保存模型元数据和用户偏好，不持久化明文模型 API key。正常保存配置时，旧的明文 key 会从本地配置中移除。

模型档案可以分别分配给文本聊天、视觉、语音识别和语音合成。空的角色选择会被保留，因此用户可以有意禁用某项能力，而不会被应用静默回退到另一个模型档案。

### 多模态图片

如需向模型发送图片，请配置一个带有 `vision` 能力的模型档案，并将其选为多模态档案。配置的端点必须支持 OpenAI 兼容的图片输入。如果提供商返回 `No endpoints found that support image input` 这类错误，afuOS 会将其显示为不支持视觉端点的错误，而不是泛化为普通模型失败。

当会话包含图片附件时，afuOS 会通过选中的视觉档案发起请求，并将图片作为 OpenAI 兼容内容片段发送。纯文本会话会继续使用选中的文本档案。

当助手回复包含标准 Markdown 图片语法时，助手生成或链接的图片会被显示出来：

```md
![description](https://example.com/image.png)
```

只要本地图片路径是应用可访问的有效绝对路径，也可以在桌面应用中渲染。

OpenAI 风格的非文本内容数组会在显示前被规范化。文本片段会渲染为文本，图片片段会被转换成 Markdown 图片标签，以便在聊天中可见。

### 语音权限

语音输入需要 macOS 麦克风和语音识别权限。如果语音识别被拒绝，请在 macOS 系统设置 > 隐私与安全性 > 麦克风和语音识别中允许 `afuos`，然后重启应用。

### 本地权限

设置中包含低风险 shell 命令和低风险浏览器操作的独立授权开关。低风险 shell 命令包括 `pwd`、`ls`、`date`、创建桌面文件夹等只读或低影响操作。会删除数据、修改系统设置、运行下载脚本或访问禁区路径的命令仍然需要确认或会被拒绝。

高风险 shell 命令永远不会被设置开关或已记住的权限规则自动允许。这包括包含 shell 管道、输入/输出重定向、命令替换、破坏性文件操作、系统控制命令、下载后执行模式、凭据工具，以及触碰禁区路径的命令。

禁区路径会被应用到直接文件目标、shell 命令路径、解码后的 `file://` URL、递归 `find` 范围和全局元数据搜索。如果递归命令会包含禁区路径，即使更宽泛的父路径曾被允许，该命令也会被拒绝。

保存的权限规则是按目标生效的，并且只用于低风险操作。如果后续操作被判定为高风险，它仍然需要用户明确确认。

### Skills 和 MCP

Skill 注册表会保存本地 skill 路径和信任状态。受信任且启用的 skill 可以将其 `SKILL.md` 内容注入模型上下文。未受信任的 skill 在用户明确信任之前只会暴露名称和路径元数据。

MCP 服务器存储在本地注册表中，并通过解析后的 argv 启动，而不是通过 shell 启动。包含 shell 控制字符的服务器命令会被拒绝。MCP 工具可以在应用中检查、调用、编辑、删除和清空。

MCP 工具调用使用与本地操作相同的权限模型。只读或低风险工具可以按特定 server/tool 组合记住授权；高风险工具始终需要确认。工具输出中的图片、文本资源或音频会尽可能转换成聊天中可见的内容。

发送给模型的 MCP 命令元数据会对 API key、bearer token、password、secret 以及常见的 `--token value` 参数形式做脱敏。

## 隐私

afuOS 是一款本地优先的桌面应用。本地操作、会话历史、图片附件元数据、日志、权限规则和记忆数据都存储在用户机器的本地 SQLite 数据库中。模型请求只会发送到用户配置的提供商端点。

日志会脱敏 bearer token、API key、password、secret 和查询字符串凭据。剪贴板、备忘录和提醒事项内容不会写入日志。Shell 日志只记录可执行文件摘要，不存储完整命令行；MCP 日志只记录 server/tool 目标，不记录原始参数。
